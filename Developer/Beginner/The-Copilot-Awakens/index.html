Part 1

What Is GitHub Copilot?

Let's start at the beginning. What exactly is GitHub Copilot?

Your AI Pair Programmer

  • GitHub Copilot is an AI-powered code assistant that lives inside your editor
  • Trained on billions of lines of public code - understands patterns, APIs, and idioms across dozens of languages
  • Suggests whole lines, entire functions, tests, documentation, and even terminal commands
  • Works in VS Code, JetBrains IDEs, Neovim, Visual Studio, and github.com
Think of GitHub Copilot as an AI pair programmer that sits right inside your editor. It's not a separate app you switch to - it's woven into the tools you already use every day. It was trained on billions of lines of publicly available code, so it understands patterns, APIs, and idioms across dozens of programming languages. It can suggest entire lines of code, complete functions, tests, documentation - even terminal commands. And it's not limited to VS Code. It works in JetBrains, Neovim, Visual Studio, and even directly on github.com. Wherever you code, Copilot can be right there with you.

How Copilot Works

  • Context gathering - Copilot reads your open file, neighboring tabs, comments, and function signatures
  • Model inference - Sends context to a large language model (LLM) hosted by GitHub
  • Suggestion delivery - Returns completions as grey "ghost text" in your editor, or full answers in chat
  • Your code stays yours - Copilot Business and Enterprise do not retain prompts or suggestions for model training
So how does it actually work? There are three steps. First, context gathering - Copilot looks at the file you have open, the neighboring tabs, your comments, function signatures, and variable names. It's building a picture of what you're working on. Second, model inference - it sends that context to a large language model hosted by GitHub, and the model generates a completion. Third, suggestion delivery - that completion comes back to your editor as either grey "ghost text" right inline, or as a full response in Chat. Now, here's the important privacy point, especially if you're in an enterprise environment: on the Business and Enterprise plans, GitHub does not retain your prompts or suggestions for model training. Your code stays yours. And to answer the question I know some of you are thinking - no, Copilot does not see your entire codebase. It sends a limited context window, not the full repository.

Part 2

Getting Started

Alright, enough theory - let's get Copilot up and running. If you have VS Code open, now's the time to follow along.

Installing & Activating Copilot

  • Open VS Code and go to Extensions (Ctrl+Shift+X)
  • Search for "GitHub Copilot Chat" and click Install
  • The inline completions engine is built into the Chat extension - one install covers everything
  • Sign in with your GitHub account when prompted
  • Verify activation: look for the Copilot icon in the status bar (bottom right)
Install the extension
First step: open VS Code and press Ctrl+Shift+X to open the Extensions panel. Search for "GitHub Copilot Chat" - you'll see it right at the top. Click Install. The extension used to be split into two - one for inline completions and one for chat - but the autocomplete engine has been rolled into the Chat extension. Just install "GitHub Copilot Chat" and you get everything. Once it's installed, VS Code will prompt you to sign in with your GitHub account. Go ahead and do that. After you sign in, look down at the bottom-right corner of VS Code - you should see a small Copilot icon in the status bar. If it's there and not greyed out, you're good to go.

Signing In & Verifying Your License

  • Click the Copilot status bar icon → "Sign in to GitHub"
  • Authorize the VS Code GitHub extension in your browser
  • If your org uses Copilot Business/Enterprise, ensure your admin has granted you a seat
  • Run GitHub Copilot: Check Status from the Command Palette to confirm you're active
  • Troubleshooting: check firewall rules, proxy settings, or SSO requirements
Finish Setup
If Copilot isn't activated yet, click that status bar icon and select "Sign in to GitHub." It'll open your browser for authorization - just click Approve and come back to VS Code. If your organization uses Copilot Business or Enterprise, make sure your admin has assigned you a seat - otherwise the sign-in will succeed but Copilot won't activate. A handy trick: open the Command Palette with Ctrl+Shift+P and run "GitHub Copilot: Check Status." That'll confirm whether everything is connected. If you hit problems, the most common culprits are corporate firewalls blocking the connection, proxy settings that need configuring, or SSO tokens that haven't been authorized for your organization.
Part 3

Inline Completions

Now we're getting to the good stuff. Inline completions are the feature most developers encounter first, and for many people, it's the moment Copilot goes from "interesting" to "I can't code without this." Let's see how it works.

Ghost Text - Your New Superpower

  • As you type, Copilot shows grey "ghost text" - predicted completions inline
  • Press Tab to accept the full suggestion
  • Press Ctrl+→ to accept word-by-word
  • Press Esc or keep typing to dismiss and get a new suggestion
  • Use Alt+] / Alt+[ to cycle through alternative suggestions
Ghost text and hotkeys
Here's what it looks like. As you type, Copilot shows grey "ghost text" - these are predicted completions appearing right inline in your editor. To accept the entire suggestion, just press Tab. But here's a pro tip that most people don't know about: press Ctrl+Right Arrow to accept just one word at a time. This is incredibly useful when the suggestion is mostly right but you want to tweak a few things as you go. If the suggestion isn't what you want, press Escape to dismiss it, or just keep typing - Copilot will generate a new suggestion based on what you've written. And if you want to see alternative suggestions, use Alt+] to cycle forward and Alt+[ to cycle back. That word-by-word acceptance with Ctrl+Right Arrow - seriously, learn that one. It's the most underused and most valuable shortcut in the whole Copilot toolkit.

Getting Better Suggestions

  • Write a comment first - describe the function in plain English, then let Copilot generate it
  • Meaningful names - calculateShippingCost gives better results than doStuff
  • Open related files - Copilot uses neighboring tabs as context
  • Type signatures and JSDoc - the more structure you give, the better the completions
Ghost text completing from a comment
Now, the quality of suggestions depends a lot on the context you give Copilot. Here are four techniques that make a real difference. First, write a comment before you write the code. Describe what the function should do in plain English, press Enter, and watch Copilot generate the implementation. This "comment-first" approach is one of the most effective patterns. Second, use meaningful names. A function called "calculateShippingCost" gives Copilot much better signals than one called "doStuff." Your variable names and function names are essentially prompts. Third, open related files in neighboring tabs. Copilot looks at your other open tabs for context, so if you're writing a function that calls an API, having the API definition open in another tab helps a lot. And fourth, add type signatures, JSDoc comments, or any other structural hints. The more structure you provide, the better the completions will be.
Part 4

Copilot Chat

Next up is Copilot Chat - the conversational interface. A lot of developers use inline completions every day but haven't explored Chat yet, so even if you've been using Copilot for a while, this part might be new to you.

Conversational AI in Your Editor

  • Open Copilot Chat with Ctrl+Shift+I (or click the chat icon in the sidebar)
  • Ask questions in natural language - "How do I read a file in Node?"
  • Copilot responds with explanations, code snippets, and follow-up suggestions
  • Conversations are context-aware - Copilot sees your open file and selection
  • Use inline chat (Ctrl+I) for quick edits right in the editor
Open Chat (Ctrl+Alt+I)
There are actually two chat surfaces in VS Code. The first is the sidebar chat panel - press Ctrl+Shift+I to open it. This is your full conversational interface. You can ask questions in plain English, like "How do I read a file in Node?" and Copilot will respond with explanations, code snippets, and follow-up suggestions. The second is inline chat - press Ctrl+I, and a small input box opens right inside the editor. This is great for quick, targeted edits like "rename this variable to camelCase" or "wrap this in a try-catch block." Both surfaces are context-aware. Copilot can see the file you have open and any text you've selected, so the more precise your context, the better the response. Try selecting a block of code before asking a question - it makes a big difference.

Useful Chat Commands

  • /explain - Explain selected code in plain language
  • /fix - Suggest a fix for problems in the selection or file
  • /tests - Generate unit tests for the selected code
  • /doc - Generate documentation comments
  • @workspace - Ask questions about your entire project, not just the active file
Chat interface
Copilot Chat comes with some built-in commands that are really handy. The /explain command takes your selected code and breaks it down in plain English - perfect when you're working in an unfamiliar codebase. The /fix command analyzes a selection and proposes corrections, which can save you a lot of debugging time. The /tests command generates unit tests that match the structure and framework of your project. And /doc generates documentation comments. But the one I want to highlight is @workspace. When you use @workspace, Copilot doesn't just look at the current file - it indexes your entire project. You can ask things like "Where is the database connection configured?" or "Which files import this module?" and get accurate answers. To see the full list of available commands, just type a slash in the chat input and browse.
Part 5

Configuring Your Workspace

Now let's talk about configuration. This is especially relevant if you're working on a team and want to standardize how Copilot behaves across everyone's environment.

Key Copilot Settings

  • github.copilot.enable - Toggle Copilot on/off per language: {"python": true, "markdown": false}
  • editor.inlineSuggest.enabled - Master switch for inline ghost text
  • github.copilot.chat.localeOverride - Force chat responses in a specific language
  • Use Workspace Settings (.vscode/settings.json) to enforce team-wide defaults
// .vscode/settings.json
{
  "github.copilot.enable": {
    "*": true,
    "plaintext": false,
    "markdown": false
  }
}
Let me walk you through the key settings. Open Settings with Ctrl+Comma and search for "copilot." The most useful setting is "github.copilot.enable" - this is a per-language toggle. You can turn Copilot on for Python and JavaScript but off for Markdown or plain text, where it might not add much value. There's also "editor.inlineSuggest.enabled," which is the master switch for ghost text. And if you need chat responses in a specific language, "github.copilot.chat.localeOverride" lets you force that. Here's the team play: put these settings in a .vscode/settings.json file in your repository and commit it. Now every developer who clones the repo gets the same configuration automatically. No more "works on my machine" for Copilot settings.

Custom Instructions

  • Create a .github/copilot-instructions.md file in your repo
  • Copilot reads these instructions and applies them to every interaction
  • Great for enforcing coding standards: "Always use TypeScript strict mode", "Prefer async/await over callbacks"
  • Team consistency - every developer gets the same AI guidance from day one
This feature is a game-changer for teams. You can create a file called copilot-instructions.md in the .github directory of any repository, and Copilot will read it and apply those instructions to every single interaction in that project. Think about that for a second. You can write things like "Always use TypeScript strict mode," "Follow our internal API naming conventions," or "Prefer async/await over raw Promises" - and Copilot will actually follow them. Every developer on your team gets the same AI guidance from day one, without having to remember to type it into every chat. It's like codifying your team's institutional knowledge into a file that the AI actually respects. Live Demo

Copilot in Action

Alright, it's demo time! Let's switch over to VS Code and see Copilot in action. We're going to write a real function from scratch, then use Chat to improve it.

Demo: Writing a Function with Copilot

  • Create a new file utils.js
  • Write a comment: // Parse a CSV string into an array of objects
  • Watch Copilot generate the full function
  • Accept with Tab, then ask Chat to add error handling
// Parse a CSV string into an array of objects
// using the first row as headers
function parseCSV(csvString) {
  const lines = csvString.trim().split('\n');
  const headers = lines[0].split(',').map(h => h.trim());
  return lines.slice(1).map(line => {
    const values = line.split(',').map(v => v.trim());
    return headers.reduce((obj, header, i) => {
      obj[header] = values[i];
      return obj;
    }, {});
  });
}
Here's what we're going to do. We'll create a new file called utils.js. Then we'll type a comment: "Parse a CSV string into an array of objects, using the first row as headers." And then we just wait. Copilot reads that comment and starts generating a complete implementation as ghost text. See it? It's using split to break the string into lines, extracting headers from the first row, and then mapping over the remaining rows to create objects. Pretty impressive for a two-line comment. Now I'll press Tab to accept the full suggestion. Your suggestion might look a little different from what's on screen - that's totally normal. Large language models are nondeterministic, so you might get a slightly different approach. That's actually a feature, not a bug. Now that we have our function, let's ask Copilot to make it better.

Demo: Using Chat to Refactor

  • Select the function → open inline chat (Ctrl+I)
  • Type: "Add error handling for malformed CSV input"
  • Review the diff - Copilot shows additions in green, removals in red
  • Click Accept or Discard
  • Then ask: "Generate unit tests for this function"
Let's select the parseCSV function and press Ctrl+I to open inline chat. Now I'll type: "Add error handling for malformed CSV input." Watch what happens - Copilot shows a diff, with additions in green and removals in red. You always have the final say: click Accept to apply the changes, or Discard to throw them away. I'll accept this one - it's added some nice defensive checks. Now let's go one step further. I'll open the sidebar chat with Ctrl+Shift+I and type "/tests" to generate unit tests. Look at that - Copilot generated test cases covering normal inputs, edge cases, and error conditions. This is the full workflow: generate the code, refine it with Chat, and test it - all without leaving your editor. Generate, refactor, test, done.

Tips for Getting the Most from Copilot

  • Be specific - "Write a Python function that validates an email address using regex" beats "write code"
  • Iterate - Accept a suggestion, then refine it with follow-up prompts
  • Review everything - AI suggestions are starting points, not final answers
  • Use context - Open related files, add comments, use meaningful names
  • Learn the shortcuts - Tab, Ctrl+→, Alt+], Ctrl+I, Ctrl+Shift+I
Before we wrap up, here are five tips that will make a huge difference in your Copilot experience. First, be specific. "Write a Python function that validates an email address using regex" will get you much better results than "write code." The more specific your prompt, the more specific the output. Second, iterate. Don't expect perfection on the first try - accept a suggestion, then refine it with follow-up prompts. Copilot is a collaborator, not a vending machine. Third - and this is important - review everything. AI suggestions are starting points, not final answers. They can contain bugs, use outdated APIs, or follow patterns that don't match your team's standards. You're the pilot; Copilot is your co-pilot. Fourth, use context. Open related files, write descriptive comments, use meaningful variable names. The more context Copilot has, the better it performs. And fifth, learn the shortcuts. Tab, Ctrl+Right Arrow, Alt+], Ctrl+I, Ctrl+Shift+I - memorize these and they'll become second nature within a day.
