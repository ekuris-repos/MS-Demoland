Part 1

What is VS Code?

Transition into the first section. Ask the audience: "How many of you have used VS Code before?" This helps gauge the room. Reassure anyone who has not used it that VS Code is designed to be approachable for beginners while being powerful enough for advanced developers.

Editor vs. IDE: What's the Difference?

Text Editor

A lightweight tool for writing and editing code. Think Notepad or Sublime Text. Fast to open, minimal features out of the box.

IDE (Integrated Development Environment)

A full-featured environment with built-in debugging, build tools, source control, and more. Think Visual Studio or IntelliJ.

Explain the spectrum between text editors and full IDEs. VS Code sits in a unique sweet spot: it starts as a lightweight editor but can be extended into a full IDE through its extension ecosystem. This is one of the key reasons it has become the most popular development tool in the world, according to the Stack Overflow Developer Survey.

What Makes VS Code Special?

  • Free and open source from Microsoft
  • Cross-platform: Windows, macOS, and Linux
  • Lightweight yet powerful; starts up in seconds
  • Built-in Git integration for source control
  • Integrated Terminal so you never leave the editor
  • IntelliSense for smart code completions
  • Extensions Marketplace with 50,000+ extensions
Highlight each point briefly. The key takeaway is that VS Code gives you the speed of a text editor with the power of an IDE. It is the number one editor worldwide, used by millions of developers daily. Mention that the Extensions Marketplace is what truly sets it apart; you can customize VS Code to fit any language, framework, or workflow.

The Extensions Ecosystem

  • Extensions add language support, themes, debuggers, and tools
  • Popular extensions: Python, ESLint, Prettier, Docker, GitLens
  • Install directly from VS Code with one click
  • Extensions are managed through the Extensions panel (Ctrl+Shift+X)
  • Organizations can recommend extensions for consistency across teams
Emphasize that extensions are what make VS Code endlessly adaptable. You are not locked into one language or workflow. If someone on the team uses Python and another uses JavaScript, both can customize VS Code for their needs. The Extensions Marketplace is browsable in-editor and on the web. Later in this session, we will install our first extension: GitHub Copilot.
Part 2

Installing and Navigating VS Code

Transition to the hands-on portion. If participants are following along, now is the time to make sure everyone has VS Code installed. Walk through the installation steps on the next slide. If the audience already has VS Code installed, you can move through the installation slide quickly.

Installing VS Code

  1. Go to https://code.visualstudio.com
  2. Click the download button for your operating system
  3. Run the installer and accept the defaults
  4. Launch VS Code after installation completes

Tip: Check "Add to PATH" during installation on Windows so you can open VS Code from any terminal with the code command.

Walk through each step. On Windows, remind participants to check the "Add to PATH" option. On macOS, they may need to open VS Code and run "Shell Command: Install 'code' command in PATH" from the Command Palette. On Linux, VS Code is available via Snap, apt, or rpm. Keep this quick if everyone already has it installed.

The VS Code Interface: Key Areas

  • Activity Bar (left edge): Switch between Explorer, Search, Source Control, Run & Debug, and Extensions
  • Side Bar: Displays the contents of the selected Activity Bar view
  • Editor Group (center): Where you write and view code; supports tabs and split views
  • Panel (bottom): Terminal, Output, Problems, and Debug Console
  • Status Bar (bottom edge): Language mode, line/column info, branch name, errors, and warnings
Use your own VS Code window as a live visual if possible. Point to each area as you describe it. The Activity Bar is the vertical strip of icons on the far left. The Side Bar changes depending on which icon is selected. The Editor Group is the main workspace. The Panel at the bottom is where the integrated terminal lives. The Status Bar shows contextual information about the current file and workspace.

The Explorer Panel

  • Open a folder or workspace: File → Open Folder
  • The Explorer shows your project's file tree
  • Create, rename, and delete files directly in the Explorer
  • Drag and drop to move files between folders
  • Use Ctrl+P (Quick Open) to jump to any file by name
The Explorer is the first view most people see in VS Code. Explain that VS Code works best when you open an entire folder rather than individual files. This gives you the full project context, which becomes important when we start using Copilot. Quick Open (Ctrl+P) is one of the most useful shortcuts; it lets you type a filename and jump straight to it without clicking through the tree.

The Integrated Terminal

  • Open with Ctrl+` (backtick) or View → Terminal
  • Run commands without leaving VS Code
  • Supports multiple terminal instances and split terminals
  • Automatically opens in your workspace root directory
  • Supports PowerShell, Bash, Command Prompt, and more
The integrated terminal eliminates the need to switch between VS Code and a separate terminal window. You can run build commands, start servers, and interact with Git all from within the editor. Mention that you can have multiple terminals open at once and even split them side by side. This is especially useful when running a development server in one terminal and executing commands in another.

The Command Palette

  • Open with Ctrl+Shift+P (or F1)
  • Search and run any VS Code command by name
  • Access settings, change themes, install extensions, and more
  • Faster than navigating menus once you know the command name
  • Pro tip: type > for commands, or remove it to search files
The Command Palette is the single most important feature to learn in VS Code. If you forget where a setting is or how to do something, open the Command Palette and start typing. It is a universal search for every action VS Code can perform. Demonstrate it live: press Ctrl+Shift+P, type "theme," and show how you can change the color theme. This shows the power of the Command Palette in a visual, memorable way.

The Extensions Panel

  • Open with Ctrl+Shift+X
  • Search for extensions by name, category, or tag
  • Click Install to add an extension instantly
  • View extension details, ratings, and download counts
  • Extensions may require a reload to activate
Walk the audience through the Extensions panel. Show how to search for an extension, view its details page, and install it. Mention that extensions can be disabled or uninstalled at any time if they are not needed. This is where we will install GitHub Copilot shortly. Point out that the extensions icon in the Activity Bar will show a badge when updates are available.

Quick Recap: VS Code Essentials

Key Shortcuts

  • Ctrl+P: Quick Open
  • Ctrl+`: Toggle Terminal
  • Ctrl+Shift+X: Extensions

Key Concepts

  • Open folders, not individual files
  • Extensions customize everything
  • The Command Palette is your best friend
  • The terminal is built right in

Part 3

What is GitHub Copilot?

Transition to the second major topic. Build some excitement here. GitHub Copilot is one of the most transformative tools to arrive in the developer ecosystem in recent years. Let the audience know that what they are about to see will fundamentally change how they think about writing code.

GitHub Copilot: Your AI Pair Programmer

  • An AI-powered coding assistant that lives inside VS Code
  • Trained on billions of lines of public code
  • Suggests code completions as you type (inline suggestions)
  • Answers coding questions through Copilot Chat
  • Supports dozens of programming languages and frameworks
Explain the "pair programmer" metaphor. Just like a human pair programmer, Copilot watches what you are working on, understands the context, and offers suggestions. It does not write your code for you; instead, it accelerates your workflow by handling boilerplate, suggesting implementations, and helping you explore unfamiliar APIs. It works with Python, JavaScript, TypeScript, Java, C#, Go, and many more languages.

How Does Copilot Work?

  • Copilot reads the context of your current file, open tabs, and workspace
  • It sends this context to a large language model (LLM) in the cloud
  • The model generates suggestions based on patterns and best practices
  • Suggestions appear as gray "ghost text" in your editor
  • You decide what to accept, modify, or ignore
Stress that Copilot is a tool, not a replacement. The developer remains in control at all times. Every suggestion is just that: a suggestion. You press Tab to accept it, or you keep typing to dismiss it. The quality of suggestions improves when Copilot has more context, which is why opening a full folder (not just a single file) matters. Copilot also respects your coding style and adapts to your project over time.

Copilot Plans at a Glance

Copilot Free

Limited completions and chat interactions per month. Great for trying out Copilot with a GitHub account.

Copilot Pro / Business / Enterprise

Unlimited completions, advanced chat features, organization-wide policy controls, and knowledge bases (Enterprise).

Briefly cover the plan tiers so people know what they have access to. Most enterprise attendees will be on Copilot Business or Enterprise, which means unlimited completions and full chat access. If anyone is on the free tier, they can still follow along; they just have monthly usage limits. The key point is that all plans include inline completions and Copilot Chat. Enterprise adds features like knowledge bases and admin controls.
Part 4

Installing and Signing In to Copilot

Now we get hands-on. If the audience is following along, make sure they have VS Code open and are ready to install the extension. This is usually the most exciting part for newcomers because they see Copilot come to life for the first time.

Installing the GitHub Copilot Extension

  1. Open the Extensions panel: Ctrl+Shift+X
  2. Search for "GitHub Copilot"
  3. Click Install on the "GitHub Copilot" extension by GitHub
  4. This also installs GitHub Copilot Chat automatically
  5. Wait for the installation to complete (a few seconds)
Walk through this step by step. Point out that there is one official extension called "GitHub Copilot" published by GitHub. Installing it gives you both inline completions and the Chat panel. There is no need to install separate extensions for each feature. If participants already have Copilot installed, they can skip this step and make sure their extension is up to date.

Signing In and Verifying Your License

  1. After installation, VS Code prompts you to sign in to GitHub
  2. Click "Sign in to GitHub" in the notification or status bar
  3. A browser window opens for authentication
  4. Authorize VS Code to access your GitHub account
  5. Return to VS Code; the Copilot icon appears in the status bar

Verification: Look for the Copilot icon (a small sparkle) in the bottom status bar. If it is active, you are ready to go.

This is where people sometimes get stuck. If the sign-in prompt does not appear, tell them to open the Command Palette (Ctrl+Shift+P) and search for "GitHub Copilot: Sign In." Common issues include corporate firewalls blocking the OAuth redirect or the user not having a Copilot license assigned. For enterprise environments, the organization admin must enable Copilot for the user's account. Make sure everyone sees the Copilot icon in the status bar before proceeding.
Part 5

Your First Copilot Experience

This is the highlight of the session. We will demonstrate two core Copilot features: inline completions (ghost text) and Copilot Chat. If possible, switch to a live VS Code window for these demos. Seeing Copilot in action is far more impactful than reading about it on slides.

Inline Completions: Ghost Text

  • Start typing code in any supported language
  • Copilot suggests completions as grayed-out "ghost text"
  • Press Tab to accept the full suggestion
  • Press Esc to dismiss it
  • Use Alt+] and Alt+[ to cycle through alternative suggestions
Demonstrate this live if possible. Open a new Python or JavaScript file and start typing a function name like "def calculate_area" or "function calculateArea." Pause and show the ghost text appearing. Accept it with Tab. Then show how to cycle through alternatives. Emphasize that Copilot works best when you provide good context: meaningful variable names, descriptive comments, and clear function signatures.

Ghost Text in Action

# Type a descriptive function name and Copilot completes the rest

def calculate_rectangle_area(length, width):
    """Calculate the area of a rectangle."""
    return length * width    # ← Copilot suggested this line

def calculate_circle_area(radius):
    """Calculate the area of a circle."""
    import math
    return math.pi * radius ** 2    # ← And this entire function
Use this slide to show what inline completions look like in practice. In a live demo, you would type just the function name and docstring, then let Copilot fill in the implementation. Point out that Copilot understood the pattern from the first function and predicted that you would want a similar function for a circle. This pattern recognition is one of Copilot's greatest strengths.

Copilot Chat: Ask Anything

  • Open Copilot Chat with Ctrl+Shift+I
  • Ask questions in natural language: "How do I read a CSV file in Python?"
  • Get code examples, explanations, and best practices
  • Chat understands your open files and workspace context
  • Use @workspace to ask questions about your entire project
Copilot Chat is like having a knowledgeable colleague available at all times. You can ask it to explain code you do not understand, generate boilerplate, write tests, fix bugs, or suggest refactoring approaches. Demonstrate by asking a question about the current file. Show how the response includes code snippets that can be inserted directly into the editor. Mention that @workspace lets Copilot search across all files in your project, not just the currently open file.

Inline Chat: Edit Code in Place

  • Press Ctrl+I to open Inline Chat directly in the editor
  • Type a natural language instruction: "Add error handling" or "Refactor this to use async/await"
  • Copilot modifies the selected code in place
  • Review the diff and click Accept or Discard
  • Great for quick, targeted edits without switching to the Chat panel
Inline Chat is one of the most practical features for day-to-day coding. Select a block of code, press Ctrl+I, and type what you want to change. Copilot shows you a diff of the proposed edit, and you can accept or reject it. This is faster than copying code into Chat and pasting it back. Demonstrate by selecting a simple function and asking Copilot to "add input validation" or "convert this to a list comprehension."

Tips for Getting Better Suggestions

  • Use descriptive names: Clear function and variable names give Copilot better context
  • Write comments first: A comment describing what you want helps Copilot predict the implementation
  • Open related files: Copilot uses open tabs as context; keep relevant files open
  • Be specific in Chat: "Write a Python function that sorts a list of dictionaries by the 'name' key" works better than "sort a list"
  • Iterate: If the first suggestion is not right, refine your prompt or try cycling through alternatives
This slide is about building good habits from the start. The quality of Copilot's output is directly related to the quality of the input you provide. Think of it as a conversation: the more context and clarity you give, the better the response. Encourage participants to experiment with different prompt styles and see how the suggestions change. This skill of giving good prompts is explored in depth in the "Prompts, Instructions, Skills, and Agents" course later in the track.

Common First-Day Questions

  • "Is Copilot reading my private code?" Copilot processes context in your editor to generate suggestions. Enterprise plans include data privacy protections; your code is not used to train models.
  • "Can I use Copilot for any language?" Copilot supports most popular languages. Suggestion quality varies; it excels in Python, JavaScript, TypeScript, Java, and C#.
  • "What if the suggestion is wrong?" Always review suggestions before accepting them. Copilot is a tool, not an oracle. You are responsible for the code you ship.
These are the three questions that come up in almost every introductory session. Address them proactively to build trust. On privacy: Copilot Business and Enterprise do not retain prompts or suggestions and do not use your code for training. On language support: it works with virtually all languages, but the depth of suggestions depends on the training data. On correctness: Copilot accelerates your workflow, but you remain the decision-maker. Testing and code review still matter.
